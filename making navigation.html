Making buttons with Fireworks

This is the navigation bar made up of buttons created with Fireworks

Steps to make buttons

  1. set up your canvas size according to the size of your navigation bar
  2. draw your buttons (usually rectangle)
  3. with the filter option, add an "Inner Bevel" filter to create a look of an buttons
  4. save it as an symbol
  5. drag the number of buttons needed for the navigation bar
  6. edit the buttons by double clicking it
  7. change how it looks when it is over and under (copy up image than change filter option)
  8. align the buttons evenly
  9. trim your canvas to make it a perfect fit to crop unnecesary white spaces around the buttons

Using the buttons in Dreamweaver

  1. Save the image as a html in dreamweaver with the "export" option
  2. In a tempelete document, create an div at the place you want your buttons to show on your page.
  3. Click on insert on the top navigation bar and choose "Fireworks html".
  4. Select your buttons file and it should show up in the div you created
  5. Then link your buttons to pages on your file
  6. To recreate the hover effect, first copy the <script> code shown in the browser preview into the head of the tempelete.
  7. Then change the link of the images by removing the ../ in the "onmouseover= '../image/" link (use the find and replace option).

Compare and Contrast between graphic navigation and text navigation

Advatages of Graphic Navigation:

  1. The final product is more attractive and more able to "call to action"
  2. It is easier to create as every step is very visuallized -- you can see what each step does to the final product right away
  3. The look of the product is easy to change and has a lot of variety.
  4. It can have more animation which keeps the audience entertained and

Disadvantages of Graphic Navigation:

  1. It is a more time consuming process, as you need to first create it with Fireworks and transfer it into Dreamweaver.
  2. The formatting and coding is more complicated which might counfues us when we try to change parts of the code.
  3. There might be synchronizing problem between the Firework document and when it's embeddedin Dreamweaver -- you might have to recreate the buttons again.
  4. Graphic Navigation might be a distraction if you don't have much CSS coded for the page.

Advantages of Word Navigation:

  1. They are easy to create in which it requires less steps
  2. It is able to show you which link you've already visited, which lessen the chance of going back to a page that doesn't have the information you wanted.
  3. It's simplicity would not distract the audience from the content of the page.
  4. We don't have to worry about synchronizing between two applications.

Disadvantages of Word Navigation

  1. The audience might found text link boring and lost interest in visiting the site.
  2. During the creating process, the changes you made might not be instatly visualized.
  3. There's nothing much to change in the look of the navigation. (less variety)

Discussion:

I think graphic navigation is the better for our website since we would like to excite our audience and inspire them to learn about kinematics. Also, the background of the website is rather fancy, I think text navitgation would not be very eye-cathing, causing a failed "call-to-action".
